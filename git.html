O que é o GitHub?

O GitHub é uma plataforma de hospedagem de código-fonte com controle de versão usando Git. Ele permite que você colabore com outras pessoas em projetos, gerencie versões, e compartilhe seu trabalho com o mundo.

Neste guia, você vai aprender desde a criação da conta até o envio de um pull request. Tudo ilustrado com fluxogramas, imagens e exemplos práticos.

Passo a Passo

1Criar uma conta

Acesse github.com e clique em Sign up. Escolha um nome de usuário, e-mail e senha. Verifique seu e-mail e pronto! Você já tem um perfil no GitHub.

2Criar um repositório

No canto superior direito, clique no + e escolha New repository. Dê um nome, uma descrição, escolha se será público ou privado e marque Add a README file. Clique em Create repository.

3Clonar o repositório

No seu computador, abra o terminal e execute: git clone https://github.com/seu-usuario/repositorio.git Isso baixa uma cópia local do projeto para você trabalhar.

4Adicionar e commitar

Adicione arquivos ao diretório clonado. Depois use: git add . e git commit -m "mensagem". O commit salva as alterações localmente com uma descrição.

5Enviar para o GitHub (push)

Para enviar seus commits para o repositório remoto, use: git push origin main (ou master). Agora seu código está online e visível para quem tiver permissão.

6Pull Request (colaboração)

Se você está colaborando, faça um fork do repositório, crie uma branch, faça alterações e abra um pull request. O dono do repositório pode revisar e mesclar suas contribuições.

Dicas e Explicações

📌 O que é Git? Git é um sistema de controle de versão distribuído. Ele rastreia alterações em arquivos e permite que várias pessoas trabalhem no mesmo projeto sem conflitos.

📌 Por que GitHub? O GitHub adiciona uma camada social e colaborativa ao Git. Você pode compartilhar projetos, contribuir com projetos open source e construir um portfólio.

📌 Branch (ramo): Uma branch é uma linha de desenvolvimento independente. Use git branch nome para criar e git checkout nome para trocar. A branch principal geralmente é main.

📌 Commit: Um commit é como um "checkpoint" no seu código. Sempre escreva mensagens claras, como "Adiciona função de login".

📌 Push e Pull: push envia seus commits para o repositório remoto; pull baixa as alterações de outras pessoas para sua máquina.

📌 Pull Request (PR): É um pedido para que o dono do repositório aceite suas alterações. É a base da colaboração no GitHub. Revise, discuta e depois mescle!

📌 Fork: Cria uma cópia de um repositório alheio para sua conta. Você pode modificar e depois abrir um PR para o repositório original.
